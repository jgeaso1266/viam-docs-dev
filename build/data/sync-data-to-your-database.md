## What You'll Learn

- How to configure direct data capture to your own MongoDB cluster
- How to set up egress patterns from Viam's cloud to external databases
- How to architect Viam as an ingestion layer with your database as the backend

## What Problem This Solves

You may need captured data in your own database — for integration with existing systems, custom retention policies, regulatory requirements, or to avoid vendor lock-in. This block shows how to route captured data to your own MongoDB cluster while still using Viam's capture and sync infrastructure.

## Components Needed

- A machine with data capture configured (from Capture and Sync Data)
- An external MongoDB cluster (MongoDB Atlas or self-hosted)
